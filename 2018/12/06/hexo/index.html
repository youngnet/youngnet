利用hexo搭建博客

发表于 2018-12-06 | 更新于 2019-05-29 | 分类于 博客搭建系列

现在互联网中有很多博客的平台，做的也都很不错。不过一次看到通过github搭建个人博客的教程，然后觉得既然是前端开发弄个有点不一样的博客还是可以的。然后就开始所有各种资料了，最后选择了hexo，当然原因肯定是有的。

  • 首先hexo编写文章可以使用markdown 个人非常喜欢的一种写作方式
  • 其次hexo是基于node+webpack搭建的编译系统对于前端开发人员来说非常利于上手同时能够做一些定制话的东西
  • 最后有很多的主题可以供选择所以对于外观党赶紧上手吧！

环境配置与安装

  • 安装hexo之前的依赖环境
    1. node —- hexo的使用都是基于node来进行的所以需要提前安装node.js（安装过的可以略过）
    2. git —- 如果你想要使用非默认主题那么你大概需要安装git 或者利用git来管理你的博客文件在多台电脑上进行操作

      这时候可以开始安装hexo了 只需要在你的电脑终端中使用 npm install hexo-cli -g 安装工作就大功告成啦 ！

编写博客部分

  • 生成一个基础博客

    进入到一个自己想要创建博客的文件夹 终端执行

  • 常用指令
    generate

  1. hexo generate

    生成静态文件。

  2. hexo g

  3. hexo publish [layout]

    发表草稿。

  1. hexo server

    启动服务器。默认情况下，访问网址为： http://localhost:4000/。

  2. hexo deploy

    • 部署网站。 -g, --generate 部署之前预先生成静态文件

  3. hexo render [file2] …

    • 渲染文件。
    • 参数 描述 -o, --output 设置输出路径

  1. hexo migrate

    • 从其他博客系统 迁移内容。

  2. hexo clean

    • 清除缓存文件 (db.json) 和已生成的静态文件 (public)。
    • 在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

  3. hexo list

    • 列出网站资料。

  4. hexo version

    • 显示 Hexo 版本。
  1. hexo –safe 安全模式
    • 在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

  1. hexo –debug 调试模式

    • 在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并 提交调试信息到 GitHub。

  2. hexo –silent 简洁模式

    • 隐藏终端信息。

  3. hexo –config custom.yml 自定义配置文件的路径

    • 自定义配置文件的路径，执行后将不再使用 _config.yml。

  4. hexo –draft 显示草稿

    • 显示 source/_drafts 文件夹中的草稿文章。
  1. hexo –cwd /path/to/cwd 自定义 CWD
    • 自定义当前工作目录（Current working directory）的路径。
